# Contribution Guidelines

Thank you for your interest in contributing to the Redwood Burl Protection Toolkit! This document outlines the process for contributing to ensure consistency and effective collaboration.

## How to Contribute

All new information, reports, and changes should be submitted via **Issues** and **Pull Requests** to ensure proper tracking and review.

### 1. Reporting Incidents or Suggesting Tasks
- Use the **Issues** tab to create a new issue.
- Provide a clear title and a detailed description.
- Use the available labels (`incident-report`, `public-awareness`, `legislative-track`, `verification-needed`) to categorize the issue.
- Include relevant details such as date, location, description, and any required actions.

### 2. Making Changes to Documentation or Code
- Fork the repository (if you are not a direct collaborator).
- Create a new branch for your changes (e.g., `add-california-burl-protection-act`).
- Make your modifications in your branch.
- Commit your changes with a descriptive commit message.
- Open a **Pull Request** (PR) from your branch to the `main` branch.
- In the PR description, reference any related issues by number (e.g., `Closes #1`).
- Wait for review and address any feedback before merging.

### 3. Review Process
- At least one other team member should review each pull request.
- Ensure that new content is accurate and appropriately sourced.
- For incident reports, verification may be required before merging.

### 4. Code of Conduct
- Be respectful and constructive in all communications.
- Focus on the shared goal of protecting redwood trees and their burls.
- Do not share sensitive or personally identifiable information in public issues.

By following these guidelines, we can maintain a productive and organized collaborative environment.
